# Asset provenance and release clearance

Audit snapshot: 2026-07-23, repository revision
`b8e194a0e519e2dd3232b9843293af0c87f8f16a`, plus the current working-tree
removal of `public/sprites/_raw_char_easyface.png`.

This file records what the repository proves about the origin of shipped media.
It is not a declaration that every asset is commercially cleared. An asset is
release-cleared only when the evidence described below has been collected and
reviewed for the exact file included in the release.

**Current release conclusion:** the media set is not cleared for publication.
The unresolved **Hold** entries below remain launch blockers regardless of
whether the app builds, the legal pages are configured, D1 is migrated, or
Play Console accepts an upload. Only the rights owner or an authorized reviewer
can supply and approve the missing authorship, license, likeness, audio, and
listing-art evidence.

Status meanings:

- **Conditional pass**: the repository shows an in-project, reproducible origin
  and no third-party source material was found, but the maintainer must still
  confirm authorship.
- **Hold**: do not publish the asset until rights evidence is attached or the
  asset is replaced with one that has evidence.
- **Exclude**: the file is not intended to ship and must remain outside
  `public/`, Android resources, and the final app bundle.

## What is and is not shipped

Vite copies every file under `public/` into the web payload, which Capacitor
then packages into Android. A file can therefore ship even when no code
references it. The final audit must inspect the built web payload and the
signed AAB/APK, not only search source imports.

The following are working/source artifacts rather than release assets unless a
build step explicitly copies them:

- `assets/`: icon masters and intermediate icon components.
- `tools/blender/output/`: ignored Blender scenes, previews, blends, and GLBs.
- `devlog/`: development screenshots and visual checkpoints.
- `scripts/`: generators and post-processing tools.

Do not move raw generations, reference photos, preview sheets, or license
documents containing personal information into `public/`.

## Evidence required before publication

Create one evidence record per asset or clearly defined asset family. Sensitive
contracts, receipts, identity records, and provider account exports may live in
a secured release archive outside Git; this file should then record the archive
reference. Never commit API keys, OAuth tokens, cookies, or private account
credentials.

Each evidence record should contain:

1. Final relative path, asset title, creator/source, creation or acquisition
   date, and SHA-256 of the exact release binary.
2. Editable source or original recording plus the tool/export settings used to
   create the shipped derivative.
3. The license, assignment, or provider terms that applied on the acquisition
   or generation date, saved as a dated PDF or screenshot with its source URL.
4. Proof that commercial mobile-app use, advertising, in-app purchases,
   worldwide distribution, updates, and required modification/redistribution
   are permitted.
5. Any receipt, invoice, provider job ID/result metadata, prompt and seed, or
   signed work-for-hire/copyright assignment.
6. Attribution, notice, share-alike, trademark, or other obligations and where
   the release satisfies them.
7. A likeness/publicity review for every depicted or referenced person. Obtain
   a release when needed; “AI generated,” “original,” “meme-inspired,” or
   stylized/pixelated is not by itself clearance.
8. For audio, separate proof covering the composition, master recording,
   performers, samples, synchronization, and distribution rights.
9. Reviewer name, decision, review date, and any geographic/term restrictions.
